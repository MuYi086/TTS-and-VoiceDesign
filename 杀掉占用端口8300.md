# 杀掉占用端口 8300

适用场景：运行 `bash ./start.sh` 时出现：

```text
ERROR: [Errno 98] error while attempting to bind on address ('0.0.0.0', 8300): address already in use
```

这表示 8300 端口已经有进程在监听。常见原因是旧的 `python api.py` / `conda run ... python api.py` 后端还在运行。

## 1. 先确认是不是已有后端可用

先不要急着杀进程，检查 8300 上是否已经是当前后端：

```bash
curl -s http://127.0.0.1:8300/v1/health | python -m json.tool
```

如果能返回健康检查，并且里面的模型路径、`indextts_ready`、`cuda` 都正常，可以直接复用这个服务，不需要再启动一次。

如果接口无响应、返回异常，或者你明确要重启后端，再继续下面步骤。

## 2. 找到占用 8300 的进程

优先用 `ss`：

```bash
sudo ss -ltnp 'sport = :8300'
```

你会看到类似：

```text
LISTEN 0 2048 0.0.0.0:8300 0.0.0.0:* users:(("python",pid=731328,fd=6))
```

记下 `pid=` 后面的数字，比如 `731328`。

如果 `ss` 看不到进程名，换用：

```bash
sudo lsof -nP -iTCP:8300 -sTCP:LISTEN
```

或：

```bash
sudo fuser -v 8300/tcp
```

## 3. 查看进程链

`conda run` 启动时通常会有一条进程链，例如：

```text
conda run --no-capture-output -n unitale-tts-local python api.py
  bash /tmp/tmpxxxx
    python api.py
```

查看相关进程：

```bash
ps -fp <PID>
ps -eo pid,ppid,stat,etime,cmd | grep -E 'api.py|unitale-tts-local|8300'
```

把 `<PID>` 换成第 2 步查到的 PID。

## 4. 温和停止

先对监听端口的 Python 进程发正常退出信号：

```bash
kill <PID>
sleep 2
```

再检查端口是否释放：

```bash
sudo ss -ltnp 'sport = :8300'
```

如果没有输出，说明端口已经释放。

## 5. 如果还没释放，杀整条后端进程链

如果 `ps` 里还能看到 `conda run`、临时 `bash`、`python api.py`，把这些 PID 一起停掉：

```bash
kill <CONDA_PID> <BASH_PID> <PYTHON_PID>
sleep 2
```

仍然不退出时再强制：

```bash
kill -9 <CONDA_PID> <BASH_PID> <PYTHON_PID>
```

不要使用 `pkill python` 这类宽泛命令，容易误杀其他 Python 任务。

## 6. 确认 GPU 显存也释放

端口释放后，再看 GPU：

```bash
nvidia-smi
```

正常情况下，`Processes` 区域不应该再出现旧的 `python api.py`。

如果 `ps -fp <PID>` 已经查不到这个进程，但 `nvidia-smi` 还显示它占显存，通常是 WSL/NVIDIA 驱动残留上下文。处理方式：

1. 关闭正在使用 WSL 的终端和 VS Code 窗口。
2. 在 Windows PowerShell 里执行：

```powershell
wsl --shutdown
```

3. 重新打开 WSL，再执行：

```bash
nvidia-smi
```

## 7. 重新启动后端

确认 8300 和 GPU 都干净后：

```bash
cd ~/github/TTS-and-VoiceDesign
bash ./start.sh
```

启动成功后验证：

```bash
curl -s http://127.0.0.1:8300/v1/health | python -m json.tool
```

看到 `code: 200`、`indextts_ready: true`、`cuda: true` 后，再回 Unitale 前端操作。

## 快速命令

如果你确认 8300 只会被本项目后端占用，可以用下面的快速流程：

```bash
sudo ss -ltnp 'sport = :8300'
ps -eo pid,ppid,stat,etime,cmd | grep -E 'api.py|unitale-tts-local|8300'
kill <PID>
sleep 2
sudo ss -ltnp 'sport = :8300'
nvidia-smi
```
