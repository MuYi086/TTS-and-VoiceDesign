# MOSS-SoundEffect v2.0 声效提示词说明

这份规范用于让上游大模型把小说片段转换为可直接传给 `MOSS-SoundEffect-v2.0` 的**非语言声效**提示词。模型支持中文和英文提示词，输出为 48 kHz 音频，单次时长最长 30 秒。每条提示词对应一次声效生成；若场景有多个独立声源，分别生成后由后处理按时间轴混音。

## 先分清职责

- 只描述可听见的环境声、动作声、动物声、器物声或非歌词音乐片段。
- 人物说话、旁白、台词、姓名、心理活动和剧情解释交给 TTS 或其他链路，**不要**写进声效提示词。
- `seconds` 是脚本参数，不要把“生成 10 秒”写在 `prompt` 中。
- 单条优先只有一个主体声源；环境底噪可以作为陪衬。两种以上同等突出的声音应拆分为多条。
- 只保留小说中明确或为场景氛围必要的声音，不要凭空添加音乐、掌声、雷雨等。

## 提示词写法

按下面的顺序，写成一条完整、具体、自然的中文或英文句子：

`主体声源 + 动作/状态 + 材质或细节 + 空间环境 + 距离/视角 + 节奏、强弱与时间变化`

应优先给出的信息：

- **主体与动作**：木门缓慢打开、皮靴踩过积雪、雨点打在玻璃上。
- **质感**：生锈、厚重、清脆、沉闷、湿润、沙沙、金属摩擦。
- **空间**：空旷走廊、安静公园、狭窄石巷、车内、远处山谷。
- **视角和距离**：近距离、门外不远处、远处、由远及近、左侧掠过。
- **时间演变**：开始稀疏后渐密、持续而均匀、三次短促敲击后停下、逐渐远去。

避免使用抽象评价（如“很有电影感”“令人恐惧”“高级”）替代具体声音，也避免“不要有人声”这类否定句；直接省去不需要的元素即可。

## 上游大模型的输出协议

### 直接调用模式（一个声效）

当调用方只需生成一个音效时，上游模型必须只输出下面 JSON 的 `prompt` 文本，不要加标题、解释、Markdown 或参数：

```json
{"prompt":"深夜的旧木门被缓慢推开，门轴发出低沉、略带生锈的连续吱呀声，安静的室内近距离收音。"}
```

调用方取得 `prompt` 字段后，直接赋值给测试脚本中的 `PROMPT`。

### 小说场景规划模式（多个声效）

若输入是一段小说，输出严格 JSON，不要补充说明。`prompt` 是唯一传给 MOSS 模型的文本；`start_seconds` 和 `duration_seconds` 仅用于调用方排程和混音，不能传给 `pipe()`。一次生成的 `duration_seconds` 必须在 `(0, 30]` 内。

```json
{
  "sound_effects": [
    {
      "id": "sfx_001",
      "start_seconds": 0.0,
      "duration_seconds": 6.0,
      "priority": "background",
      "prompt": "清晨安静的林间小径，近处零星鸟鸣清脆，远处有轻微树叶在微风中沙沙作响，氛围宁静而自然。"
    },
    {
      "id": "sfx_002",
      "start_seconds": 1.5,
      "duration_seconds": 3.0,
      "priority": "foreground",
      "prompt": "皮靴踩在潮湿落叶上的脚步声，从中等距离缓慢靠近，踩踏声柔软清晰，节奏稳定。"
    }
  ]
}
```

字段约束：

| 字段 | 规则 |
| --- | --- |
| `id` | 当前场景内唯一，推荐 `sfx_001` 格式。 |
| `start_seconds` | 大于等于 0 的数字；仅供混音排程。 |
| `duration_seconds` | 大于 0 且不超过 30；调用脚本时映射为 `SECONDS`。 |
| `priority` | `foreground`、`background` 或 `transition`。 |
| `prompt` | 一条具体的中英文非语言声效描述；不可为空、不可包含台词或 JSON 以外的控制符。 |

## 从小说提取时的决策规则

1. 按叙事时间顺序找出实际发生的可闻事件和持续环境声。
2. 把持续场景氛围标为 `background`，把动作、动物叫声、敲门等瞬态事件标为 `foreground`，场景切换用 `transition`。
3. 同一时刻有多个独立的主声源时拆成多条；例如“暴雨中马车驶过”可拆为雨声、车轮碾过湿石路和马蹄声。
4. 只从文本可推知的内容补足合理声学细节。比如“推开生锈铁门”可增加金属摩擦，但不能加“雷声”，除非文本交代雷雨。
5. 每条以最小必要时长呈现完整动作：短促事件一般 1–5 秒；连续环境一般 5–30 秒。不要为了凑时长重复罗列同义词。

## 示例

小说输入：

> 雨后的巷子湿漉漉的。阿宁撑着伞，踩过积水，远处一辆公交车轰鸣着驶离。

合格输出：

```json
{
  "sound_effects": [
    {
      "id": "sfx_001",
      "start_seconds": 0.0,
      "duration_seconds": 8.0,
      "priority": "background",
      "prompt": "雨后狭窄石巷的安静环境声，屋檐偶尔滴下水珠，湿润的空气中有轻微远处城市底噪。"
    },
    {
      "id": "sfx_002",
      "start_seconds": 1.0,
      "duration_seconds": 3.0,
      "priority": "foreground",
      "prompt": "雨伞下的脚步踩进浅水坑，发出两三次清晰而湿润的溅水声，近距离收音。"
    },
    {
      "id": "sfx_003",
      "start_seconds": 3.0,
      "duration_seconds": 6.0,
      "priority": "background",
      "prompt": "远处城市街道上一辆公交车发动机低沉轰鸣，逐渐加速并向远处驶离，声音慢慢减弱。"
    }
  ]
}
```

不合格：`一段悲伤又电影感十足的雨夜音乐，阿宁伤心地走着。`

原因：它混入了情绪、人物行为解释和未经文本要求的音乐，且没有可控的具体声源。
